# DPDP Act 2023 — Compliance Checklist

**Use this checklist to track your organisation's readiness for the Digital Personal Data Protection Act, 2023.**

---

## Phase 1: Discovery & Data Mapping

| # | Item | Status | Notes |
|---|------|--------|-------|
| 1.1 | Identify all personal data collected by your organisation | ☐ | Include customer, employee, vendor data |
| 1.2 | Map data flows across systems, departments, and third parties | ☐ | Collection → storage → processing → sharing → deletion |
| 1.3 | Classify data types (regular personal data vs. children's data) | ☐ | Children = under 18 |
| 1.4 | Identify lawful basis for each processing activity | ☐ | Consent or legitimate use (Section 7) |
| 1.5 | Document data retention periods and deletion schedules | ☐ | Minimum 1 year for logs (Rule 8) |
| 1.6 | Identify cross-border data transfers and destinations | ☐ | Check notified restricted territories |
| 1.7 | Determine if your organisation qualifies as an SDF | ☐ | Significant Data Fiduciary criteria |

---

## Phase 2: Consent & Notice

| # | Item | Status | Notes |
|---|------|--------|-------|
| 2.1 | Draft clear, plain-language consent requests | ☐ | Independent, understandable, itemised |
| 2.2 | Implement consent collection with affirmative action | ☐ | Pre-ticked boxes not allowed |
| 2.3 | Ensure withdrawal mechanism is as easy as giving consent | ☐ | One-click withdrawal comparable to opt-in |
| 2.4 | Prepare notices in English and Eighth Schedule languages | ☐ | Option to access in either language |
| 2.5 | Include DPO/contact information in all notices | ☐ | Business contact details |
| 2.6 | Update consent for pre-existing data principals | ☐ | "As soon as reasonably practicable" |
| 2.7 | Implement Consent Manager integration if applicable | ☐ | Registered with Board |

---

## Phase 3: Security Safeguards

| # | Item | Status | Notes |
|---|------|--------|-------|
| 3.1 | Implement encryption/obfuscation/masking for personal data | ☐ | At rest and in transit |
| 3.2 | Set up role-based access controls (RBAC) | ☐ | Least privilege principle |
| 3.3 | Enable audit logging and monitoring | ☐ | Monitor access and modifications |
| 3.4 | Establish backup and recovery procedures | ☐ | Regular tested backups |
| 3.5 | Review and update Data Processor contracts | ☐ | Valid written agreements in place |
| 3.6 | Retain logs for minimum 1 year | ☐ | Per Rule 6(1)(e) |
| 3.7 | Document technical and organisational measures | ☐ | Compliance evidence |

---

## Phase 4: Breach Response

| # | Item | Status | Notes |
|---|------|--------|-------|
| 4.1 | Establish breach detection and escalation process | ☐ | Internal team + legal counsel |
| 4.2 | Prepare breach notification template for the Board | ☐ | Incident description, consequences, measures, contact |
| 4.3 | Prepare breach notification template for Data Principals | ☐ | Plain language, timely |
| 4.4 | Set up 72-hour detailed reporting capability | ☐ | Updated and detailed report to Board |
| 4.5 | Conduct tabletop breach response exercises | ☐ | Test actual readiness |
| 4.6 | Document roles and responsibilities for breach response | ☐ | Who does what, when |

---

## Phase 5: Data Subject Rights

| # | Item | Status | Notes |
|---|------|--------|-------|
| 5.1 | Set up mechanism for Data Principals to access their data | ☐ | Right to information |
| 5.2 | Implement correction/completion/update workflows | ☐ | Timely response to requests |
| 5.3 | Implement data erasure workflows | ☐ | Consent withdrawal + purpose no longer served |
| 5.4 | Establish nomination process for Data Principals | ☐ | Post-death/incapacity nominee |
| 5.5 | Build grievance redressal mechanism | ☐ | Respond within 90 days (Rule 14(3)) |
| 5.6 | Publish DPO/contact information publicly | ☐ | For complaints and questions |

---

## Phase 6: Children's Data

| # | Item | Status | Notes |
|---|------|--------|-------|
| 6.1 | Identify processing involving data of children (under 18) | ☐ | |
| 6.2 | Implement verifiable parental consent mechanism | ☐ | Per Rule 10 — age/identity verification via authorised entities |
| 6.3 | Ensure no tracking/behavioural monitoring of children | ☐ | Prohibited under Section 9(3) |
| 6.4 | Ensure no targeted advertising to children | ☐ | Prohibited under Section 9(3) |
| 6.5 | Review if exempt under Fourth Schedule | ☐ | Healthcare, education, etc. exemptions |
| 6.6 | Implement consent for persons with disabilities | ☐ | Guardian verification per Rule 11 |

---

## Phase 7: Significant Data Fiduciary (SDF) Obligations

| # | Item | Status | Notes |
|---|------|--------|-------|
| 7.1 | Appoint Data Protection Officer (based in India) | ☐ | Responsible to Board of Directors |
| 7.2 | Appoint independent data auditor | ☐ | |
| 7.3 | Conduct annual Data Protection Impact Assessment (DPIA) | ☐ | Every 12 months |
| 7.4 | Conduct annual compliance audit | ☐ | Every 12 months |
| 7.5 | Report DPIA/audit findings to the Board | ☐ | Per Rule 13 |
| 7.6 | Perform algorithmic software due diligence | ☐ | Algorithmic transparency |
| 7.7 | Assess data localisation requirements | ☐ | For specified personal data |

## Phase 8: Ongoing Compliance

| # | Item | Status | Notes |
|---|------|--------|-------|
| 8.1 | Schedule annual DPIA and audit | ☐ | |
| 8.2 | Conduct regular employee training on data protection | ☐ | All staff handling personal data |
| 8.3 | Review and update privacy notices periodically | ☐ | Reflect any changes in processing |
| 8.4 | Monitor regulatory developments (Board notifications, SDF lists) | ☐ | Stay informed |
| 8.5 | Maintain records of processing activities | ☐ | Evidence of compliance |
| 8.6 | Review cross-border transfer restrictions as notified | ☐ | Updated restricted territories |
| 8.7 | Verify third-party/vendor compliance | ☐ | Data Processor obligations |

## Key Dates

| Deadline | Event |
|----------|-------|
| **13 Nov 2026** | Consent Manager registration rules in force (Rule 4) |
| **13 May 2027** | Majority of rules in force — this is your primary go-live date |
| *Ongoing* | Monitor for SDF notifications, restricted territory lists, exemptions |

---

**How to use this checklist:**
1. Assign ownership for each item to a team member
2. Track status: Not Started → In Progress → Completed → Verified
3. Document evidence for each completed item (policy documents, system configurations, contracts)
4. Review quarterly to ensure continued compliance

*This checklist is a practical tool and does not constitute legal advice.*
